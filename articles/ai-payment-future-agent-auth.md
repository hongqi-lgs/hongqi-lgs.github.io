> AIが人間に代わって決済を行うようになると、決済システム全体の根本的なロジックが書き直される。プログラマブル口座、エージェント認可アーキテクチャ、OpenAIの決済参入——この変革は多くの人が予想するよりずっと早く訪れようとしている。

決済はすでに解決済みの問題に見える。QRコード、カード、NFC——体験は十分にスムーズだ。しかしそのスムーズさは、一つの前提の上に成り立っている。**支払いの主体は人間である**、という前提だ。

この前提が揺らぎ始めたとき、決済システム全体を基盤から作り直す必要が生じる。

## 「人間が払う」から「エージェントが払う」へ

AIアシスタントに「来週の上海出張を手配して」と言うだけで、新幹線の切符を調べ、ホテルを予約し、出発時の配車を呼んでくれる。その間、あなたはアプリを一度も開かず、パスワードを一度も入力しない。

これはSFではない。三年から五年以内に普及するインタラクションの形だ。

しかし、この利便性の中に誰も気づいていない深層の問題がある。**誰が支払ったのか？**

現行の決済インフラは、身元確認から不正検知モデルに至るまで、「操作者は人間」を前提として設計されている。顔認証、SMS認証、行動バイオメトリクス——これらの防壁はAIに対して意味をなさず、正常に動作しているAIと悪意を持って操られているAIの区別もつかない。

決済業界が直面しているのは機能のアップデートではなく、**アーキテクチャの全面的な再構築**だ。

## プログラマブル決済：お金に「ロジック」を持たせる

従来の決済は一つのアクションだ。AからBへ金が動く。プログラマブル決済はプログラムだ。**条件Xが満たされたとき、AからBへ金が動く。しかも目的Yにのみ使用可能**。

### 三つの並行する技術路線

**スマートコントラクト**——イーサリアムのコントラクトは任意に複雑な決済ロジックを記述できる。コードが契約であり、実行に信頼できる仲介者は必要ない。ただしオラクル問題や一般ユーザーの参入障壁といった課題もある。

**中央銀行デジタル通貨**——特にデジタル人民元が注目に値する。人民銀行はすでに「条件付き決済」機能を公開テストしている。使途限定、有効期限設定、場所のロックといった機能だ。これは国家レベルのインフラが推進するプログラマブル通貨であり、他国には類似したものが存在しない。

**オープンバンキング**——欧州のPSD2/PSD3規制の枠組みは銀行にAPIの開放を義務付けている。IETFのRAR（Rich Authorization Requests）標準は、「口座へのアクセス」という粗粒度の認可を「今日の18時までに店舗Aに150元以下を支払う」という細粒度のロジックにアップグレードしている。

### 本当の価値：信頼コストの消滅

B2B取引における支払いサイト（ネット30、ネット60）は信頼コストの外部化だ。プログラマブル決済は契約条項を直接決済ロジックとして記述できる。商品受領かつ品質検査通過 → 自動的に支払いが実行される。支払いサイトという概念は経済学的に不要になる。

より深い変化は**成果報酬型ビジネスモデルの普及**だ。広告費は実際のコンバージョンに対して支払われ、SaaSは実際の成果に対して課金される。AIが結果を検証し、プログラマブル決済が実行する。

## エージェント認可アーキテクチャ：三層モデル

核心的な矛盾は一つだ。ユーザーは利便性を求める（エージェントの自律的な操作）が、コントロールを失いたくない（お金が勝手に使われてはならない）。

### アイデンティティ層

ユーザーが自分のマスター鍵で委任証明書に署名してエージェントに発行する。証明書には有効期限、認可スコープ、上限金額が明記され、再委任不可とする。W3CのDIDとVerifiable Credentials標準が最も有力な技術的基盤だ。

```json
{
  "type": "AgentPaymentCredential",
  "issuer": "did:user:alice_master_key",
  "subject": "did:agent:claude_instance_xyz",
  "claims": {
    "maxAmountPerTx": 200,
    "maxDailyAmount": 1000,
    "allowedCategories": ["food", "transport"],
    "validUntil": "2026-04-30T23:59:59Z",
    "delegatable": false
  }
}
```

### 権限層

五つの次元で精密なコントロールが必要だ。金額上限、許可カテゴリ、時間制限、頻度制限、確認ルール（閾値超過時は人的承認を要求）。

### 実行層

**クーリングウィンドウ**——エージェントが開始した決済は30秒から5分の待機状態に入り、ユーザーはキャンセルできる。

**サーキットブレーカー**——過去の行動パターンから一定閾値以上逸脱した場合、エージェントの決済権限を自動的に凍結する。

### マルチエージェントのネスト問題

あなたのAIアシスタントが旅行プラットフォームのAIを呼び出し、そのAIがホテルのAIを呼び出し、最終的にホテルのAIが決済を実行する場合、**権限は縮小のみ可能、拡大不可**という原則が重要だ。委任のたびにスコープは狭まるのみで、被害を制御可能な範囲に収められる。

## アイデンティティ層争奪：次の10年の決済入口

表面上は市場シェアの争いだが、本質は**エージェントのアイデンティティ層のコントロール**だ。

現時点で最も有利なのはAppleだ。Secure Enclave + Apple Pay + Apple Intelligenceは自然なアイデンティティ-決済の閉じたループを形成している。

OpenAIとAnthropicが最大の変数だ。

**OpenAI**の動機は強い。エージェントがすべての取引に深く関与しながら、現行システムでは収益を得られない。World ID（虹彩スキャン）とWorldcoinの組み合わせは、アイデンティティ + AI + 決済の完整な閉じたループへの布石だ。

**Anthropic**の戦略はより間接的だ。MCP（Model Context Protocol）で標準を定義し、決済ツールの認可拡張が追加されれば、自分で決済を行う必要なく標準税を徴収できる。

## 規制の行方

- **中国**：強い管理から入る。デジタル人民元を基盤に、AI決済ホワイトリスト制度が設けられる可能性が高い。
- **米国**：市場が先走って規制が後を追う。イノベーション空間は大きい。
- **EU**：立法してから展開する。AI ActとPSD3の連動で最も厳格なフレームワークが構築される。

すべての規制フレームワークが直面する核心的な問いは同じだ。**AIが人間に代わって支払ったとき、責任は誰にあるか？**

## おわりに

プログラマブル決済、エージェント認可アーキテクチャ、AIアイデンティティ層——この三つの方向は同一の終点を指している。

**お金は意図を持つようになる。** ただの数字ではなく、自分が何に使えるか、誰が引き金を引けるか、どのような条件で流れるかを知っている。

これは効率への巨大な向上であり、同時に前例のない支配力の集中だ。このルールを設計した者が次の時代の金融インフラを支配する。
